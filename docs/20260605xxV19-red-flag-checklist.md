# 20260605xxV19 Red Flag Checklist

## Purpose

Identify weak signals after an advisor or buyer review.

## Red Flags

Check any that apply.

```text
[ ] Reviewer liked the concept but named no real use case.
[ ] Reviewer framed this only as generic AI governance.
[ ] Reviewer thought this was AI judging AI.
[ ] Reviewer did not understand the Trust Decision Record.
[ ] Reviewer did not understand why evidence is central.
[ ] Reviewer saw no urgency.
[ ] Reviewer could not name a decision owner.
[ ] Reviewer could not name a budget path.
[ ] Reviewer could not identify a pilot candidate.
[ ] Reviewer focused only on dashboards or visuals.
[ ] Reviewer expected autonomous approval or remediation.
[ ] Reviewer treated SOC 2 as automatic vendor approval.
[ ] Reviewer treated framework mapping as compliance proof.
```

## Most Important Red Flag

```text
Reviewer cannot name a decision being made with AI that needs defensible evidence.
```

## Action Guidance

### One or Two Red Flags

Continue discovery. The reviewer may still be useful.

### Three to Five Red Flags

Treat as weak buying signal. Use feedback for product clarity, not sales priority.

### Six or More Red Flags

Do not prioritize as near-term buyer unless they introduce a stronger stakeholder.

## Notes

Red flags are not failures. They help identify whether the market message, use case, or buyer path needs adjustment.
