## Delayed-feedback handling

When feedback arrives after the reviewed build:

1. Record when it was originally expressed, when known.
2. Identify the exact build or artifact the source saw.
3. Compare the original problem with the current product state.
4. Preserve durable insight without restoring obsolete scope.
5. Accept, constrain, defer, reject, supersede, or request owner clarification.
6. Update the controlling requirement and identify the superseded item.
7. Never treat praise, enthusiasm, or arrival date alone as authority.
