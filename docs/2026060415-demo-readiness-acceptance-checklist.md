# 2026060415 ATDR Demo Readiness Acceptance Checklist

## Purpose

This checklist defines when CyberShield AI Decision Assurance is ready to show to a serious reviewer without apologizing for prototype roughness.

## Required Entry Points

- [ ] `launch.html` loads
- [ ] `atdr.html` loads
- [ ] `atdr-smoke.html` loads
- [ ] Launch page explains the product in plain English
- [ ] Legacy Executive OS remains available but secondary

## Demo Router

- [ ] Vendor Risk demo selectable
- [ ] Security demo selectable
- [ ] Compliance demo selectable
- [ ] Selected demo visibly loads
- [ ] Button language is clear
- [ ] Active scenario is obvious to a viewer

## Vendor Risk Demo

- [ ] AI recommendation appears in Intake
- [ ] Claims screen shows atomized vendor-risk claims
- [ ] SOC 2 existence and SOC 2 scope are separate claims
- [ ] Encryption claim is separate
- [ ] Customer data access claim is separate
- [ ] Low-risk conclusion is separate
- [ ] Unsupported approval leap is visible
- [ ] Missing evidence is visible
- [ ] Risk If Wrong is High or appropriately conservative
- [ ] Human review is required

## Security Demo

- [ ] AI recommendation appears in Intake
- [ ] Vulnerability acceptance claim is visible
- [ ] Exploit likelihood claim is visible
- [ ] Asset exposure claim is visible
- [ ] Compensating control claim is visible
- [ ] Remediation deferral claim is visible
- [ ] Unsupported low-risk leap is visible
- [ ] Security reviewer path is visible

## Compliance Demo

- [ ] AI recommendation appears in Intake
- [ ] NIST mapping claim is visible
- [ ] Policy sufficiency claim is visible
- [ ] Implementation evidence claim is visible
- [ ] Test evidence claim is visible
- [ ] Compliance conclusion claim is visible
- [ ] Required warning language appears for framework mappings
- [ ] Compliance posture is not overclaimed

## Decision Brief

- [ ] Can I act on this? is obvious
- [ ] Recommended action is obvious
- [ ] Risk If Wrong is obvious
- [ ] Confidence band is obvious
- [ ] Primary blocker is visible
- [ ] Required reviewer path is visible
- [ ] Export posture is visible

## Demo Coach

- [ ] Demo Coach button appears
- [ ] Coach opens and closes
- [ ] Coach changes by workflow stage
- [ ] Coach gives a presenter-friendly talk track
- [ ] Coach does not appear in print output

## Evidence Workbench

- [ ] Synthetic evidence appears
- [ ] Local file upload works in browser
- [ ] Evidence notes can be added
- [ ] Uploaded evidence is described as session-only
- [ ] Weak, stale, self-attested, or incomplete evidence is not treated as final proof

## Human Review

- [ ] Reviewer role is visible
- [ ] Human selected action can be saved
- [ ] Reviewer note can be saved
- [ ] Residual risk note can be saved
- [ ] Human decision remains separate from CyberShield recommendation

## Export

- [ ] JSON export downloads
- [ ] Browser print opens
- [ ] Executive brief is readable
- [ ] Signature block appears
- [ ] Limitations appear
- [ ] Record metadata appears
- [ ] CyberShield and Maximum Justice Cybersecurity branding appears
- [ ] Dr. Max Justice appears as human-in-the-loop authority where appropriate

## Smoke Test

- [ ] Smoke test analyzes all three demos
- [ ] Smoke test shows pass/fail summary
- [ ] Schema validation result is visible
- [ ] JSON parse check passes
- [ ] Framework warning check passes

## No-Overclaim Checks

The demo must not imply:

- [ ] Compliance certification
- [ ] Legal determination
- [ ] Audit opinion
- [ ] Production enforcement
- [ ] Autonomous approval
- [ ] Live vendor approval
- [ ] Secure enterprise storage in this static prototype
- [ ] Live model verification in this static prototype

## Demo-Ready Decision

The prototype is demo-ready when:

- [ ] The vendor-risk demo can be shown end-to-end in under ten minutes
- [ ] The exported brief can stand alone
- [ ] The presenter does not need to explain away basic UI issues
- [ ] The product story is clear: the record is the product
