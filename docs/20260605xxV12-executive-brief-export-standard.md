# 20260605xxV12 Executive Brief Export Standard

## Purpose

Define the required DOCX, PDF, and JSON export standard for the CyberShield V1 vendor-risk Trust Decision Record workflow.

## Export Philosophy

The export is a core product artifact.

It must be polished enough for:

- Executive review
- Consulting delivery
- Signature workflow
- Enterprise presentation
- Later challenge or review

If the export looks like plain text converted to PDF, it fails.

## Required Export Formats

Required:

- DOCX executive brief
- PDF executive brief
- JSON structured record

Optional:

- Markdown

## DOCX Requirement

DOCX is required because it supports:

- Opening in Microsoft Word
- Manual editing
- Signature workflow
- Final save as PDF
- Use as a consulting deliverable
- Inclusion in client-facing reports

## PDF Requirement

PDF is required because it is the final executive artifact.

The PDF should use the same structured content and visual template as the DOCX.

## JSON Requirement

JSON is required because it preserves structured record data and supports future integrations.

## Visual Standard

Recommended formatting:

- Page size: Letter
- Margins: 0.7 inch to 1.0 inch
- Body font: Aptos, Calibri, Arial, or similar
- Body font size: 10.5 pt to 11 pt
- Section headings: 13 pt to 16 pt
- Title: 18 pt to 22 pt
- Line spacing: 1.05 to 1.15
- Professional paragraph spacing
- Clean tables
- Selective bolding
- Clear subject line
- Footer with record ID, version, and page number
- CyberShield blue, graphite, black, and white palette
- Minimal red/yellow status markers only where useful

## Branding

Use the MJC/CyberShield logo asset in the header or cover area.

Prepared by line:

```text
Prepared by Maximum Justice Cybersecurity using CyberShield Decision Assurance
```

## Required Sections

1. Cover/header
2. Subject line
3. Executive Decision Brief
4. Recommendation Under Review
5. Decision Context
6. Key Findings
7. Material Claims and Evidence Table
8. Contradictory Evidence
9. Risk If Wrong
10. Confidence Band
11. Human Review Requirement
12. CyberShield Recommended Action
13. Human Decision and Override
14. Reviewer Notes
15. Decision Owner
16. Limitations
17. Signature Block
18. Export Metadata

## Export Metadata

Every export should include:

- Record ID
- Record version
- Export timestamp
- Export version
- Analysis version
- Record defensibility band
- Generated by
- File hash where practical

## Export Boundary

The executive brief does not certify compliance, approve the vendor, or replace accountable review.

It documents the evidence, gaps, risk, rationale, review path, and limitations available at the time of export.
