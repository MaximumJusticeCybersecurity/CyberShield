# 2026060415 Trust Kernel Naming Note

## Naming Decision

Within the CyberShield AI Decision Assurance demo flow, the former legacy Executive OS surface should be presented as:

```text
Trust Kernel
```

The launch page button should read:

```text
Open Trust Kernel
```

not:

```text
Open Legacy Executive OS
```

## Rationale

The ATDR demo should not make the older CyberShield surface feel obsolete or discarded.  Calling it Trust Kernel positions the existing platform as the underlying trust layer while ATDR becomes the focused decision-record workflow.

## Product Language

Use this distinction:

```text
CyberShield ATDR is the workflow.
Trust Kernel is the underlying trust surface.
The AI Trust Decision Record is the product artifact.
```

## Current Implementation

The launch page has been updated so the fourth action button reads:

```text
Open Trust Kernel
```

and still links to:

```text
index.html
```
