# Consumer Contract 1.0.0

Consumers shall pin exact package version `0.1.0` and the recorded tarball SHA-256, preserve package validation failures, and keep domain rules in a separate domain pack.  Consumers may structure, normalize, link, validate, and prepare review records.  Consumption does not grant credentials, network routes, storage, model access, external action, approval, verifier status, or permission expansion.

Consumers shall not copy the canonical source, import runtime code from raw URLs, use Git submodules as package distribution, silently substitute versions, modify package output to conceal missing evidence, or represent provenance and confidence as truth.  A major version is required for breaking interface or authority changes.  Every upgrade requires regression, rollback, and exact-integrity review.
