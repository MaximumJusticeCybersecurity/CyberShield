Vendor-Risk Guided Demo Checklist

This page is the operator checklist for controlled demos. It protects the current doctrine: vendor-risk first, record-first, no fake production claims.

Primary path: vendor-risk.htmlCapture simulated until endpoint configuredLegacy route preservedLog defects before building fixes

Route Manifest · Record Contract · Open QA Defect Log

1. Primary demo route

Open vendor-risk.html.

  • Six-step guided route loads.
  • Diagnostics show Engine OK.
  • Summary shows 10 claims, Request Evidence, High Risk If Wrong, Low confidence, Review Yes.

2. Contract and smoke checks

Open record-contract.html and vendor-risk-smoke.html.

  • Both should show GO.
  • Claims extracted should equal 10.
  • CRM payload should include structured record JSON.
  • Out-of-scope humanity case should remain Unknown confidence and Not defensible.

3. Guided flow

  • Enter first name.
  • Enter optional company.
  • Enter optional vendor.
  • Proceed through each step without using the old workbench rail.
  • Change contradiction type and confirm evidence emphasis changes.

4. Report artifact

  • Click Print / Save PDF from a non-final step.
  • Confirm output is the executive Trust Decision Record summary, not only the current step.
  • Go to final step and print again.
  • Confirm both outputs include action, risk, confidence, review, claims, candidate actions, and limitations.

5. Capture behavior

  • On final step, enter email.
  • Click Prepare CRM Payload.
  • Expected current result: simulated capture.
  • Do not claim Google Sheet capture succeeded until REPORT_CAPTURE_ENDPOINT is configured.

6. Route protection

  • route-manifest.html confirms the route doctrine.
  • atdr.html still opens the deeper workbench.
  • trust-kernel-legacy.html still opens legacy Trust Kernel.
  • Root page points users toward guided vendor-risk demo first.

Stop conditions

  • If guided route is blank, use the route fail-safe buttons.
  • If smoke route is NO-GO, do not demo externally.
  • If record contract checker is NO-GO, do not demo externally.
  • If print output is unreadable, do not use PDF as proof artifact.
  • If capture says simulated, do not tell anyone the CRM was updated.
  • If route manifest does not show vendor risk as the primary wedge, stop and fix route governance.
  • If a defect is observed, record it in QA Defect Log before building fixes.
