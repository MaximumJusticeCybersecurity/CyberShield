V60.3.23 scope:
V60.3.23 stops reactive image-path patching and creates a governed TrustMap asset intake contract. It keeps the faster V60.3.21/V60.3.22 shell and prewarm architecture, while preparing CyberShield for rebuilt Layer 1 assets with black backgrounds, uniform scale, and optional WebP variants.

Runtime import rule:
The operational layer should not statically import the full TrustMap stack during initial page load. The executive shell should load first. The full interactive TrustMap loads on demand.

Asset manifest rule:
TrustMap assets should be governed by `data/trustmap/v60-3-23-asset-manifest.json`. Current PNG files remain valid fallbacks. Future rebuilt black-background all-blue cube assets should be added through the manifest rather than hardcoded in scattered modules.

Image prewarm rule:
Do not force image prewarm into the critical startup path. Start prewarm after shell readiness. Prefer manifest paths. Warm the CyberShield Trust Kernel first, then Layer 1 PNGs and Briefing snapshot. If images remain slow, compress/resize PNG assets or add WebP/AVIF variants before adding more runtime logic.

Layer 1 asset rules:
- Layer 1 assets should not have constant red/yellow/green glow in normal state.
- Stoplight glow appears only on hover, focus, or selected/clicked state.
- Click/select glow persists until another Layer 1 asset is selected.
- All Layer 1 holographic cube images must render at the same apparent size.
- Third Parties & Vendors must not be smaller, cropped, zoomed, square-boxed, or styled differently from the others.
- The visual priority is a bright, consistent holographic cube look across all Layer 1 assets.

Future asset intake rule:
- Use one locked template for canvas size, cube scale, camera angle, lighting, and margins.
- Use black or near-black backgrounds.
- Keep base art neutral blue/white/black.
- Green/yellow/red remain UI trust-state overlays only.
- Provide web-ready PNG and preferably WebP variants.

Performance doctrine:
- Load the executive shell first.
- Do not force mobile users to pay the full TrustMap cost during startup.
- Warm TrustMap images after the shell is usable.
- Load the full interactive TrustMap only when requested.
- Prioritize TrustMap images when TrustMap is explicitly opened.
- Prefer one owned render/reapply path over stacked timers and event listeners.

Controlling scoring doctrine:
CyberShield must not show decorative scores. Every score must trace to a named, versioned model with factors, weights, evidence inputs, confidence logic, missing evidence, assumptions, risk if wrong, and action consequence.

Guiding principle:
No score without a model. No model without evidence. No evidence without traceability. No action without consequence.
