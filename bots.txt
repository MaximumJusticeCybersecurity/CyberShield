V60.3.21 scope:
V60.3.21 addresses phone QA feedback that CyberShield was slow to load and that the TrustMap still showed jitter. The heavy TrustMap stack is now dynamically imported only when the TrustMap tab or the Briefing Expand full TrustMap action is used. A lightweight placeholder appears while the TrustMap stack loads, mobile connector animations and expensive filters are reduced, and images are marked for async decode where possible.

Runtime import rule:
The operational layer should not statically import the full TrustMap stack during initial page load. The executive shell should load first. The full interactive TrustMap loads on demand.

Layer 1 asset rules:
- Layer 1 assets should not have constant red/yellow/green glow in normal state.
- Stoplight glow appears only on hover, focus, or selected/clicked state.
- Click/select glow persists until another Layer 1 asset is selected.
- All Layer 1 holographic cube images must render at the same apparent size.
- Third Parties & Vendors must not be smaller, cropped, zoomed, square-boxed, or styled differently from the others.
- The visual priority is a bright, consistent holographic cube look across all Layer 1 assets.

Performance doctrine:
- Load the executive shell first.
- Do not force mobile users to pay the full TrustMap cost during startup.
- Load the full interactive TrustMap only when requested.
- Prefer one owned render/reapply path over stacked timers and event listeners.
- If the TrustMap remains slow, consolidate connector rendering and reduce animated SVG/filter load before adding any more visual overlays.

Stoplight trust color rule:
- Green = Good / Strong / Allowed
- Yellow = Needs Verification / Constrained / Moderate
- Red = At Risk / Critical / Refused
- Blue is neutral brand/system/fiber/AI-working state, not trust-state severity.

Connector rule:
Default connectors should resemble the white-blue fiber optic line in the CyberShield Trust Kernel image. Connectors should be drawn center-point to center-point behind rendered objects. Rendered images stay above connector lines so lines appear to terminate at object perimeters. Trust-state connector highlights may use digital fiber optic green, yellow, or red depending on selected/scenario path.

Three-pane layout rule:
TrustMap dashboard has three distinct panes: left = Operational Trust Score and executive trust context, center = Enterprise Trust Map, right = selected item breakdown. All three panes must share the same top start line and bottom stop line. The right pane must not overlap or visually swallow the center map pane.

Briefing snapshot rule:
The Briefing page TrustMap Snapshot uses assets/The Trust Map.png. The snapshot is a static executive preview. The full interactive TrustMap remains on the TrustMap tab.

Controlling scoring doctrine:
CyberShield must not show decorative scores. Every score must trace to a named, versioned model with factors, weights, evidence inputs, confidence logic, missing evidence, assumptions, risk if wrong, and action consequence.

Guiding principle:
No score without a model. No model without evidence. No evidence without traceability. No action without consequence.

Prototype boundary:
The current GitHub Pages build is a lightweight static advisory prototype. It does not perform live enterprise integrations, live retrieval, live scoring, statistical validation, backend persistence, workflow automation, production enforcement, certification, or compliance determination.
