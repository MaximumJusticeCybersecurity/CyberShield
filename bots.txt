Strategic direction:
CyberShield remains MJC-owned advisory software, not standalone SaaS yet. Sandeep path is advisor / IBM practice-builder / potential internal champion. Regulated SMBs and federal-adjacent SMBs remain the commercial fallback market.

Critical UX rule:
Every visible executive object that appears as a chip, metric, card, gate, node, report, status, or control must either explain itself on hover/focus or navigate somewhere useful. No dead bubbles.

Critical architecture rule:
CyberShield must remain consolidated and performant. Do not reintroduce old phase-file module sprawl, broad MutationObserver loops, dynamic script injection, or timestamp-based runtime loading.

Prototype boundary:
The current GitHub Pages build is a lightweight static prototype. It is not yet connected to live enterprise telemetry, real identity systems, SIEM data, EDR, cloud control planes, GRC tooling, production agent enforcement, Google Sheets, Microsoft 365, or real CRM sync. Do not represent it as a fully deployed live enforcement engine.
