# AI Decision Assurance Architecture (AIDA)

AIDA is the owner-approved methodology and strategic workstream governing how CyberShield determines whether an AI-generated recommendation is trustworthy enough to act upon.

Start here:

- `docs/aida/README.md`
- `docs/aida/aida-manifesto-v1.0.md`
- `docs/aida/program-charter-and-roadmap.md`
- `docs/aida/repository-governance-and-contributor-process.md`

Core rule:

> If a proposed feature does not improve an organization's ability to determine whether an AI-generated recommendation should be trusted before action is taken, it does not belong in the current product.

AIDA is model-agnostic, vendor-neutral, and intended to remain compatible with current and future AI platforms.

This file is an entry point.  `AGENTS.md`, `SECURITY.md`, `security-policy-manifest.json`, current owner instruction, protected CyberShield requirements, and the source-of-truth hierarchy remain controlling.
